This tab enables the visualisation and editing of detected parameters (including their parsed types/values) within parsed OpenAPI files (found in the 'Parser' tab).


To effectively use the match and replace feature when assessing OpenAPI-based RESTful APIs, set valid test values in the 'Edited Value' column. The match and replace will only be applied to requests that meet all of the following conditions:

  • The BurpSuite tool to monitor/process is selected in the 'Match/Replace Scope' section of this tab.
  • The request contains at least one parameter with its name and type matching 'Parameter' and 'Type', and its value matching 'Parsed Value'.

For optimal results and accuracy, fill the 'Edited Value' column with valid test parameters (i.e., those that trigger an HTTP 200 response) before launching any scans.


Warning: Operations in the 'Parser' tab, such as clicking 'Clear item(s)' or 'Clear all' options in the contextual menu, or clicking the 'Browse/Load' button, will reset the 'Parameters' tab.

  • Any click on the 'Clear item(s)' or 'Clear all' options of the contextual menu.
  • Any click on the 'Browse/Load' button.

Known bugs (PRs are welcome):

  • Body parameters can only be formatted as 'application/x-www-form-urlencoded' due to current Burp Extender API limitations.
  • Editing the 'Edited Value' column in the 'Parameters' tab while filtering the table may result in the edited value being set to 'null'.
  • To register a change made in the 'Edited Value' column, you should press the Enter key first and then press the Escape key.
  • Deep/recursive parsing of OpenAPI Schema fields is not supported.
